**Module structure**:
```html
<!-- in shlav-a-mega.html, replace the 13 function bodies with: -->

  import * as P from './js/pure.js';
```

**Risk**: 🟢 LOW. All 13 functions are pure/read-only. Rollback = `git revert` the single commit.

**Estimated win**: ~350 lines removed from monolith (~7% reduction), new file ~380 lines.

```js

```

**Tests affected**: Whichever tests currently import FSRS wrappers. May need to pass `getState`/`save` mocks. Expect 5-10 test file updates.

**sw.js impact**: add `/js/fsrs-adapter.js`, bump version.

**Risk**: 🟢 LOW-MEDIUM. The pattern is established by `shared/fsrs.js`. Test updates are mechanical.

**Estimated win**: ~200 lines removed.

## Phase 3 — flagged, NOT recommended yet

Do not extract these until Phase 1 and 2 are stable and you've added integration tests:

- **Quiz engine** (`buildPool`, `check`, `next`, `pick`, `setFilt`, etc.) — tangled with `S`, render dispatch, and exam mode.
- **UI renderers** (`render`, `_rqmQuestion`, `_rqmControls`, `_rqmTeachBack`, `_rqmExplain`, `_rqmFooter`) — thick DOM mutations, tight coupling to `S` and pool state.
- **Feature-specific UI** (medBasket drug picker, EOL decision tree) — coupled to quiz engine UI.
- **Global `S` and `save()`** — load-bearing. Extracting these requires introducing a proper state library (Zustand/nanostores) or signals, which would violate the zero-build constraint.

## Rollback strategy

Every phase step is a single commit. Rollback = `git revert <sha> && npm test && git push`. SW cache version will bump forward (never backward) on the revert commit, so users get the rolled-back code correctly.

## Anti-patterns explicitly flagged

- **"Let's use Vite"** — No. Breaks `python -m http.server 3737` workflow, breaks CI's raw Node parsing, breaks GitHub Pages' zero-build promise.
- **"Extract and refactor in one step"** — No. Extract first (behavior-identical), refactor later in a separate commit. Makes bisect useful.
- **"Move `S` into a module"** — No. Too load-bearing; every renderer references it. Consider only if you later introduce signals or a state library, and even then: phased.
- **"Use dynamic imports for lazy-loading"** — Not yet. Adds complexity to SW caching for little win at this app size.
